## 九、建议的终极优化方案

### 方案 A：单页应用 (SPA)

将 12 个页面合并为 `app.py` 中的 12 个 Tab，只执行一次初始化。

**优点**：
- 只加载一次，后续切换不重复执行
- Session State 集中管理
- 代码维护更简单

**缺点**：
- 单文件代码量较大
- 需要重构现有页面结构

---

### 方案 B：懒加载 + 预加载

```python
# 当前页面立即加载
# 相邻页面预加载到缓存
# 远处页面按需加载
```

**优点**：
- 保持现有结构
- 渐进式优化

**缺点**：
- 缓存管理复杂
- 仍有部分重复计算

---

### 方案 C：数据层分离

```
Streamlit (展示层)
    ↓
FastAPI (API 层) ← 缓存所有计算
    ↓
SQLite (数据层)
```

**优点**：
- 前后端分离
- API 可复用
- 性能最优

**缺点**：
- 架构复杂度高
- 需要额外部署
