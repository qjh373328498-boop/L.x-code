> 版本：v1.0

## 映射规则

1. **优先级**：精确匹配 > 模糊匹配 > 上下文推断
2. **多关键词匹配**：选择权重最高的
3. **无匹配**：不触发任何 skill，正常对话

## 意图强度识别

### 强意图（系数 ×1.0）
```
- 帮我 XXX
- 我要 XXX
- 需要 XXX
- 怎么做 XXX
- 准备 XXX
```

### 弱意图（系数 ×0.6）
```
- 了解一下 XXX
- 看看 XXX
- 问一下 XXX
- XXX 是什么
```

### 无意图（系数 ×0）
```
- XXX 怎么样
- XXX 好不好
- 知道 XXX 吗
- 你好/谢谢/再见
```

## 置信度决策阈值

```json
{
  "direct_trigger": 80,
  "confirm_trigger": 60,
  "ask_confirm": 40
}
```

| 置信度 | 决策 | 操作 |
|--------|------|------|
| ≥80% | 高置信 | 直接触发 skill |
| 60-79% | 中置信 | 触发 + 告知用户 |
| 40-59% | 低置信 | 询问用户确认 |
| <40% | 不触发 | 正常对话 |

## 多意图处理

### 优先级
```
P0：明确指定 skill 名
P1：一级关键词匹配
P2：二级关键词匹配
P3：三级关键词匹配
```

### 冲突解决
```
如果多个 skill 置信度接近：
1. 询问用户确认
2. 按顺序执行
3. 选择最可能的一个
```

### 示例
```
用户："帮我写论文，需要查文献和数据分析"

识别结果：
- research-paper: 100%
- literature-search: 100%
- data-analysis: 100%

决策：全部触发，按顺序执行
```
